git配置多个SSH-Key

发表于2020-06-23|更新于2022-03-21|技术学习SSH
|字数总计:361|阅读时长:1分钟

前言

由于Coding日常爆炸，不得不考虑所有站点都进行多部署操作了。但是在其他平台最好的方式就是通过GitHub作为媒介，进行无缝同步。于是为了个人仓库看起。来不是那么混乱，于是决定在注册一个GitHub账号，专门用来存放静态页面。作为多部署的媒介（GitHub Pages太慢了，只能放弃）。

那么问题来了，我的ssh公钥已经在我的主账号添加了，小号不能在添加这个公钥。于是只能新创建一个密钥对。但是无论如何，小号都无法通过验证。

ssh-keygen -t rsa -C "examp@examp.com” -f ~/.ssh/github-gos967

解决方案

通过向百度学习，找到了一种较为妥协的方案。

修改~/.ssh/config文件，配置git信息

# 账号1登陆认证
Host github.com
    HostName github.com
    IdentityFile ~\\.ssh\\id_rsa

# 账号2的登陆认证
Host tzki.github.com
    HostName github.com
    IdentityFile ~\\.ssh\\gos967_rsa
  • Host表示别名，随便设置即可。
  • HostName表示服务器的地址。
  • IdentityFile即表示自定义文件

这里第一个设置表示链接服务器github.com（别名）这个服务器时，使用id_rsa密钥。

第二个设置表示链接服务器tzki.github.com时使用gos967_rsa密钥。

接下来只需要将小号的地址修改为git@tzki.github.com:xxx/xxx.git即可。（原地址git@github.com:xxx/xxx.git）
